🎯 C# Class & Constructor Reference Guide

Complete scenarios for instantiation and inheritance rules

Class Types in C#

Class Type Instantiable Inheritable Notes
Public Class ✅ Yes ✅ Yes Standard class, can be instantiated and inherited.
Abstract Class ❌ No ✅ Yes Must be inherited; derived class must implement abstract members.
Sealed Class ✅ Yes ❌ No Can be instantiated but cannot be inherited.
Static Class ❌ No ❌ No Cannot be instantiated or inherited; only contains static members.
Internal Class ✅ Yes* ✅ Yes* *Within same assembly only. Access limited to assembly scope.
Private Class (nested) ✅ Yes* ✅ Yes* *Within containing class only. Only accessible inside the containing class.
Partial Class ✅ Yes ✅ Yes Split across multiple files; behaves like a normal class.
Record Class (C# 9+) ✅ Yes ✅ Yes Immutable reference type with value semantics.

Constructor Scenarios in C#

Class Type Constructor Type Instantiable Inheritable Notes
Public Class Public Constructor ✅ Yes ✅ Yes Normal instantiation and inheritance.
Public Class Private Constructor ❌ No ❌ No Used for Singleton or Factory patterns.
Public Class Protected Constructor ❌ No* ✅ Yes *Outside class. Only derived classes can call it.
Public Class Internal Constructor ✅ Yes* ✅ Yes* *Only within same assembly.
Public Class Static Constructor N/A ✅ Yes Used for static initialization; runs once per type.
Abstract Class Public/Protected Constructor ❌ No ✅ Yes Derived classes can call base constructor.
Abstract Class Private Constructor ❌ No ✅ Yes* *Derived classes cannot call it; used for static factory methods.

Static Class Instance Constructor ⚠️ Error ⚠️ Error Compilation error: Static classes cannot have instance constructors.
Nested Private Class Public Constructor ✅ Yes* ✅ Yes* *Inside containing class. Scope limited to containing class.

Special Constructor Scenarios

Scenario Instantiable Inheritable Explanation
Class with ONLY private constructor(s) ❌ No ❌ No Singleton pattern / Factory pattern
Class with private + protected constructors ❌ No ✅ Yes Derived classes can use protected constructor
Class with private + public constructors ✅ Yes ✅ Yes Public constructor allows instantiation
Class with ONLY static constructor ✅ Yes* ✅ Yes *Compiler adds default public constructor (unless it's a static class)
Abstract class with ONLY private constructor ❌ No ✅ Yes* *Derived classes cannot call constructor; static factory pattern
Class with multiple constructors (overloading) ✅ Yes ✅ Yes As long as at least one is accessible
Record with positional parameters (C# 9+) ✅ Yes ✅ Yes Generates constructor + properties automatically

📝 Special Notes

  • Default Constructor: If no constructor is defined, C# provides a default public constructor (except for static classes).
  • Copy Constructor: Not built-in like C++, but can be manually defined.
  • Private Constructors in Abstract Classes: Prevent derived classes from calling that specific constructor.
  • Static Constructors: Cannot have parameters, cannot be called explicitly, and execute once per type.
  • Protected Internal: Accessible within same assembly OR derived classes (anywhere).
  • Private Protected (C# 7.2+): Accessible only in derived classes within same assembly.

Key Rules to Remember

🔒 Static Classes

No instance constructors, no inheritance, only static members allowed.

🎨 Abstract Classes

Cannot instantiate directly, but derived classes call base constructors.

🔐 Sealed Classes

Cannot inherit, but can instantiate (unless private constructor).

🏭 Private Constructors

Block external instantiation; used for Singletons/Factories.

🛡️ Protected Constructors

Prevent external instantiation, allow inheritance.

⚡ Static Constructors

Run once per type, automatically invoked, no parameters allowed.

🆕 Default Constructor

Auto-generated ONLY if no constructors are defined (except static-only).

📦 Records (C# 9+)

Reference types with built-in value semantics and primary constructors.

⚠️ Invalid Combinations (Compilation Errors)

  • abstract sealed class - Cannot combine abstract and sealed modifiers
  • Static class with instance constructor - Static classes cannot have instance members
  • Sealed class with protected constructor - Warning: Protected is meaningless (no inheritance possible)
