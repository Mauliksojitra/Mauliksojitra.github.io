Async / Await
Scale APIs without deadlocks. Async/await does not create a new thread — it frees the calling thread to do other work while waiting for I/O or long-running operations.
What async/await Really Does

When you await an operation, the current thread is released back to the thread pool. When the awaited operation completes, execution resumes — possibly on a different thread. No new thread is created.

⚠️
Never use .Result or .Wait() on async tasks in non-async code — this causes deadlocks by blocking the thread that the continuation needs to resume on.
Common Mistakes
❌ .Result / .Wait()
Blocks the calling thread. Can deadlock in ASP.NET or UI contexts.
❌ async void
Exceptions can't be caught by the caller. Only valid for event handlers.
❌ Fire & Forget
Exceptions are swallowed silently. Use _ = Task.Run() only if you truly don't care about the result.
Parallel Patterns: WhenAll & WhenAny
C# — Task.WhenAll
// Run multiple tasks in parallel — wait for ALL to complete
var task1 = FetchUserAsync(userId);
var task2 = FetchOrdersAsync(userId);
var task3 = FetchPreferencesAsync(userId);

await Task.WhenAll(task1, task2, task3);

var user    = await task1;
var orders  = await task2;
var prefs   = await task3;
C# — Task.WhenAny (timeout pattern)
var fetchTask   = FetchDataAsync();
var timeoutTask = Task.Delay(TimeSpan.FromSeconds(5));

var completed = await Task.WhenAny(fetchTask, timeoutTask);

if (completed == timeoutTask)
    throw new TimeoutException("Request timed out");
Cancellation with CancellationToken
C#
public async Task ProcessAsync(CancellationToken ct)
{
    using var cts = CancellationTokenSource.CreateLinkedTokenSource(ct);
    cts.CancelAfter(TimeSpan.FromSeconds(10));

    await SomeOperationAsync(cts.Token);
}
Concurrency Limits with SemaphoreSlim

Avoid self-DDoSing external APIs by limiting how many requests run concurrently.

C# — SemaphoreSlim
var semaphore = new SemaphoreSlim(3); // max 3 concurrent requests

async Task ProcessItemAsync(string item)
{
    await semaphore.WaitAsync();
    try   { await CallExternalApiAsync(item); }
    finally { semaphore.Release(); }
}
✅Use SemaphoreSlim (not Semaphore) in async code — it has async-compatible WaitAsync().
Interview Questions
Q: When would you use Task.Run()?
To offload CPU-bound work to the thread pool, keeping the UI thread or request thread responsive. Don't use it for I/O-bound work that already has an async API.
Q: Why is .Result dangerous?
It blocks the calling thread synchronously. If the continuation needs that same thread (SynchronizationContext), you get a deadlock.
Q: How do you limit concurrency safely?
Use SemaphoreSlim with WaitAsync(), or Channels, or custom task schedulers with a degree-of-parallelism limit.
Q: What's wrong with async void?
Exceptions thrown inside async void crash the process because the caller cannot observe them. Always use async Task instead.

BCL in Action
C#
using System;
using System.Collections.Generic;
using System.IO;

class Program
{
    static void Main()
    {
        // System namespace
        string msg = "Hello, BCL!";
        Console.WriteLine(msg.ToUpper()); // HELLO, BCL!

        // System.Collections.Generic
        List<int> numbers = new List<int> { 1, 2, 3, 4 };
        numbers.ForEach(n => Console.WriteLine(n));

        // System.IO
        File.WriteAllText("example.txt", "Written using BCL.");
        string content = File.ReadAllText("example.txt");
        Console.WriteLine(content);
    }
}

Constructors in C#
A special method automatically called when an object is created. Same name as the class, no return type. Used to initialize object data members.
1. Default Constructor

No parameters. Automatically provided by the compiler if none is defined. Initializes numerics to 0, booleans to false, reference types to null.

C#
class Geeks {
    int num;      // default: 0
    string name;  // default: null
    bool flag;    // default: false

    public Geeks() { Console.WriteLine("Constructor Called"); }
}
2. Parameterized Constructor

Takes at least one parameter — allows each instance to be initialized with different values.

C#
class Geek {
    string n; int i;

    Geek(string n, int i) { this.n = n; this.i = i; }

    public static void Main() {
        Geek o = new Geek("GFG", 1);
        Console.WriteLine($"Name = {o.n} Id = {o.i}"); // Name = GFG Id = 1
    }
}
3. Copy Constructor

Creates a new object by copying values from an existing object of the same class. C# does not provide one automatically — you write it manually.

C#
public Geeks(Geeks s) { // Copy constructor
    month = s.month;
    year  = s.year;
}

Geeks g1 = new Geeks("June", 2018);
Geeks g2 = new Geeks(g1); // g1 copied to g2
4. Private Constructor

Prevents direct instantiation. Used for the Singleton pattern and classes with only static members.

C#
public class Singleton {
    private static Singleton _instance;

    private Singleton() { } // private!

    public static Singleton Instance =>
        _instance ??= new Singleton();
}
5. Static Constructor

Invoked once per class. No parameters, no access modifier. Runs before the first instance is created or any static member is accessed.

C#
class Config {
    static Config() {
        // runs once, no params
        Console.WriteLine("Static ctor");
    }
    public Config(int i) {
        Console.WriteLine($"Instance {i}");
    }
}
📌Constructor ordering: Static → Instance. The static constructor runs once, before the first instance constructor is ever called.

Delegates
A type-safe function pointer — allows methods to be referenced and invoked dynamically. Foundation of events, callbacks, and functional-style programming in C#.
Declaration & Basic Usage
C# — Basic Delegate
// 1. Declare
public delegate void MyDelegate(string message);

// 2. Method matching the signature
public static void DisplayMessage(string msg)
    => Console.WriteLine("Message: " + msg);

// 3. Instantiate & invoke
MyDelegate del = DisplayMessage;
del("Hello from delegate!"); // Message: Hello from delegate!
Multicast Delegates

Delegates can reference multiple methods using +=. All methods are called in FIFO order.

C# — Multicast
public delegate void Notify();

public static void MethodA() => Console.WriteLine("Method A executed");
public static void MethodB() => Console.WriteLine("Method B executed");

Notify notify = MethodA;
notify += MethodB;  // chain MethodB
notify -= MethodA; // remove MethodA

notify(); // Method B executed
Delegates with Return Types
C#
public delegate int Operation(int x, int y);

public static int Add(int a, int b) => a + b;
public static int Multiply(int a, int b) => a * b;

Operation op = Add;
Console.WriteLine(op(5, 3)); // 8

op = Multiply;
Console.WriteLine(op(5, 3)); // 15
Callbacks
Pass behavior as a parameter — e.g., a custom sort comparison, progress reporter.
Event Handling
Events in C# are built on top of delegates — event MyDelegate OnAction;
Func & Action
Built-in delegate types: Func<T,TResult> and Action<T> avoid custom declarations.
💡For multicast delegates with return values, only the last method's return value is returned to the caller — previous return values are discarded.

Dictionary — Thread Safety
A static Dictionary<TKey,TValue> is not thread-safe. Multiple concurrent reads/writes cause race conditions, exceptions, and corrupted state.
Solution 1: lock (Simple & Safe)
C#
public static class Repository
{
    private static Dictionary<int, string> _data = new();
    private static readonly object _lock = new object();

    public static void AddOrUpdate(int key, string value)
    {
        lock (_lock) { _data[key] = value; }
    }

    public static string Get(int key)
    {
        lock (_lock)
            return _data.ContainsKey(key) ? _data[key] : null;
    }
}
Solution 2: ConcurrentDictionary (Recommended)

From System.Collections.Concurrent — thread-safe out of the box with better performance than manual locks in high-concurrency scenarios.

C# — ASP.NET Core Session Cache Example
using System.Collections.Concurrent;

public static class SessionRepository
{
    private static ConcurrentDictionary<string, string> _sessions = new();

    public static void AddOrUpdate(string userId, string token)
        => _sessions.AddOrUpdate(userId, token, (k, old) => token);

    public static string Get(string userId)
    {
        _sessions.TryGetValue(userId, out var token);
        return token;
    }

    public static bool Remove(string userId)
        => _sessions.TryRemove(userId, out _);
}
Solution 3: ReaderWriterLockSlim (Read-Heavy Workloads)

Allows multiple simultaneous readers, but exclusive access for writers — more scalable when reads far outnumber writes.

C#
private static ReaderWriterLockSlim _lock = new();

public static string Get(int key)
{
    _lock.EnterReadLock();   // multiple readers allowed
    try   { return _data[key]; }
    finally { _lock.ExitReadLock(); }
}

public static void Set(int key, string val)
{
    _lock.EnterWriteLock();  // exclusive write access
    try   { _data[key] = val; }
    finally { _lock.ExitWriteLock(); }
}
ApproachBest ForPerformance
lockSimple cases, low contentionGood for low traffic
ConcurrentDictionaryWeb APIs, high concurrencyBest overall
ReaderWriterLockSlimRead-heavy workloadsBest when reads dominate
ImmutableDictionaryRare updates, snapshot readsZero lock overhead for readers

Both store collections of data, but differ significantly in type safety, size flexibility, and performance.

Array Example
C#
int[] arr = new int[4];
arr[0] = 47;
arr[1] = 77;
arr[2] = 87;
arr[3] = 97;

foreach (int n in arr)
    Console.WriteLine(n);
✅Type-safe, fast, fixed size. Use for homogeneous data.
ArrayList Example
C#
ArrayList al = new ArrayList();
al.Add("Ajay");
al.Add("Ankit");
al.Add(10);
al.Add(10.10);

foreach (var item in al)
    Console.WriteLine(item);
⚠️ArrayList is legacy. Use List<T> in all modern C# code.
📌In modern C#, always prefer List<T> over ArrayList. It is generic, type-safe, and performs better because it avoids boxing/unboxing.

Garbage Collection
Automatic memory management by the CLR. Objects with no references become eligible for collection — developers don't need to manually free memory like in C/C++.
Generations
Generation 0 — Newly Allocated
All new objects start here. GC collects Gen 0 most frequently since most objects are short-lived. Survivors are promoted to Gen 1.
Generation 1 — Buffer Zone
Survivors of at least one Gen 0 collection. Acts as a buffer between short-lived and long-lived objects. Collected less often.
Generation 2 — Long-Lived
Objects surviving multiple collections: static data, caches, app-lifetime objects. Collected least frequently — expensive to scan.
Phases of Collection
1. Marking
GC scans objects, identifies all reachable (live) objects. Unreferenced objects are marked as garbage.
2. Relocating
References of live objects are updated so pointers remain valid when objects are moved during compaction.
3. Compacting
Dead objects are freed. Live objects are moved together to eliminate gaps, reducing fragmentation.
Key GC Methods
C#
// Force collection (use sparingly — impacts performance)
GC.Collect();
GC.WaitForPendingFinalizers();

// Memory usage
long before = GC.GetTotalMemory(false);
int[] arr = new int[1000];
long after = GC.GetTotalMemory(false);
Console.WriteLine($"Allocated: {after - before} bytes");

// Generation info
Console.WriteLine("Max generation: " + GC.MaxGeneration); // 2
Console.WriteLine("String gen: " + GC.GetGeneration("hello"));
✅Advantages: No manual memory management, prevents leaks, compaction reduces fragmentation.
⚠️Limitations: Non-deterministic timing, GC pauses can impact latency, finalizers run on separate thread.

Jobs & Agents
Jobs are units of work; Agents are the services that execute them. Together they power automation and scheduling in SQL Server and .NET applications.
Background Job with Hangfire
C# — Hangfire
public class EmailService
{
    public void SendEmail(string to, string subject, string body)
        => Console.WriteLine($"Email sent to {to}: {subject}");
}

// Enqueue a fire-and-track background job
BackgroundJob.Enqueue(() =>
    new EmailService().SendEmail("user@example.com", "Hello", "Test"));
Worker Service Agent (.NET BackgroundService)
C#
public class JobAgent : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        while (!ct.IsCancellationRequested)
        {
            Console.WriteLine("Agent running: " + DateTime.Now);
            await Task.Delay(5000, ct); // tick every 5 s
        }
    }
}

// Register in Program.cs
builder.Services.AddHostedService<JobAgent>();

ref & out Keywords
Both pass arguments by reference rather than by value, but they have different initialization requirements and directional semantics.
out Keyword

Used when a method needs to return multiple values. The parameter does not need to be initialized before calling — but must be assigned inside the method.

C#
static void Main() {
    int G;           // not initialized
    Sum(out G);
    Console.WriteLine(G); // 160
}

public static void Sum(out int G) {
    G = 80;
    G += G;         // must assign before return
}
💡Classic use: int.TryParse("42", out int result)
ref Keyword

The variable must be initialized before passing. Changes made inside the method are reflected in the caller — bidirectional flow.

C#
static void Main() {
    string str = "Geek"; // must be initialized
    SetValue(ref str);
    Console.WriteLine(str); // GeeksforGeeks
}

static void SetValue(ref string str1) {
    if (str1 == "Geek")
        Console.WriteLine("Hello!!Geek");
    str1 = "GeeksforGeeks"; // caller sees this
}
Key Differences
Aspectrefout
Must initialize before call✅ Yes❌ No
Must assign inside method❌ No✅ Yes
Data flow directionBidirectional (in + out)Unidirectional (out only)
Primary use caseModify caller's variableReturn multiple values

Task vs Thread
Thread is a low-level OS construct. Task is a high-level abstraction over the thread pool — with built-in support for async/await, cancellation, and error handling.
Task

From System.Threading.Tasks. Uses the thread pool. Supports async/await, cancellation tokens, and Task<T> for return values.

C# — Task
static async Task Main()
{
    Task<string> task = Task.Run(() =>
        "Hello from Task!");

    string result = await task;
    Console.WriteLine(result);
}
Thread

From System.Threading. Low-level, creates a dedicated OS thread. Full manual control over start, stop, priority, and lifecycle.

C# — Thread
static void PrintNumbers() {
    for (int i = 1; i <= 5; i++) {
        Console.WriteLine("Worker: " + i);
        Thread.Sleep(500);
    }
}

Thread t1 = new Thread(PrintNumbers);
t1.Start();

🟦Use Task for almost everything modern — simplicity, scalability, and async/await support make it the right default choice.
🟥Use Thread when you need explicit control: setting thread priority, foreground vs background, or long-running dedicated work loops.
