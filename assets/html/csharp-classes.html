What is a Class?

A class is a blueprint or template for creating objects. It bundles data (fields & properties) and behavior (methods) together into a single reusable unit.

When you create an object from a class, you're creating an instance — a real, in-memory version of that blueprint with its own set of values.

Classes are the foundation of Object-Oriented Programming (OOP) in C# and support four key pillars: Encapsulation, Inheritance, Polymorphism, and Abstraction.

Car.cs 
class Car
{
    // Fields (data)
    public string Brand;
    public int Speed;

    // Method (behavior)
    public void Drive()
    {
        Console.WriteLine(
          $"{Brand} at {Speed} km/h"
        );
    }
}

// Creating an object
Car myCar = new Car();
myCar.Brand = "Toyota";
myCar.Speed = 120;
myCar.Drive(); // Toyota at 120 km/h
// 02 — Class Types

8 Types of Classes in C#

🔷

Regular (Concrete) Class

The standard class — can be instantiated and inherited freely.

✓ Instantiate ✓ Inherit ✓ General Purpose 
class Animal
{
    public string Name;
    public void Speak()
      => Console.WriteLine($"{Name} speaks");
}
// ✅ Can create object directly
Animal a = new Animal();
🔶

Abstract Class

Cannot be instantiated. Forces derived classes to implement abstract members.

✗ Instantiate ✓ Inherit ✓ Base Template 
abstract class Shape
{
    public abstract double Area();
}
class Circle : Shape
{
    public double Radius;
    public override double Area()
      => Math.PI * Radius * Radius;
}
🔒

Sealed Class

Prevents any other class from inheriting it. Great for security or optimization.

✓ Instantiate ✗ Inherit ✓ Security 
sealed class BankAccount
{
    public double Balance { get; set; }
}

// ❌ This would cause an error:
// class Savings : BankAccount { }
📦

Static Class

Only contains static members. Cannot be instantiated or inherited. Used for utilities.

✗ Instantiate ✗ Inherit ✓ Utility 
static class MathHelper
{
    public static int Square(int x) => x * x;
    public static int Cube(int x) => x * x * x;
}
// Called without creating object:
MathHelper.Square(4); // → 16
🧩

Partial Class

Split a single class across multiple files. All parts merge at compile time.

✓ Instantiate ✓ Inherit ✓ Multi-file 
// File1.cs
partial class Employee
{
    public string Name;
}
// File2.cs
partial class Employee
{
    public void Display()
      => Console.WriteLine(Name);
}
🪆

Nested Class

A class declared inside another class. Useful for grouping tightly-coupled logic.

✓ Instantiate ✓ Inherit ✓ Encapsulated 
class Outer
{
    private int x = 10;

    class Inner  // Nested class
    {
        public void Show()
          => Console.WriteLine("Inner!");
    }
}
🏗️

Generic Class

Works with any data type using type parameters <T>. Reusable and type-safe.

✓ Instantiate ✓ Inherit ✓ Type-Safe 
class Box<T>
{
    public T Value;
    public void Show()
      => Console.WriteLine(Value);
}
Box<int> b1 = new() { Value = 42 };
Box<string> b2 = new() { Value = "Hi" };
📋

Interface

Defines a contract. Classes implementing it must provide all members. Enables polymorphism.

✗ Instantiate ✓ Implement ✓ Contract 
interface IFlyable
{
    void Fly(); // No body
}
class Bird : IFlyable
{
    public void Fly()
      => Console.WriteLine("Flying!");
}

Key Concepts

Field
A variable declared directly inside a class that stores data for the object.
Property
Provides controlled access to fields via get and set accessors.
Method
A function defined in the class that describes the behavior of its objects.
Constructor
A special method called automatically when a new object is created with new.
Destructor
Called when the object is being destroyed; used to release unmanaged resources.
Inheritance
A class can extend another using :, inheriting its members and behavior.
Encapsulation
Hiding internal details using access modifiers like private, protected, public.
Polymorphism
A method behaving differently based on the object calling it via override.
