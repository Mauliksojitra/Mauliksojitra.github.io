Machine Learning Model

A model is a program that can be used to recognize a pattern in data.

  • Model can be used to predict future behavior
  • A model can be used to Categorize something as one thing or another
  • A model can be used to Recognize people, objects and landmarks using unseen images
  • A model can be used to Understand the context of natural human text or speech

Uses

  • Predict future - traffic on specific day on specific route
  • Categorize - apple/banana
  • Recognize - people, objects and landmarks
  • Understand - natural human text

Guiding Principles for Responsible AI

1. Fairness

AI systems should treat everyone fairly and avoid affecting similarly situated groups of people in different ways.

  • Imagine a hospital emergency room AI system that prioritized patients in the order of their FICO credit score… the more money you have, the faster you see a doctor.
  • Imagine a bank loan AI system that gave more money to people named Michael and less money to people named Alice.

2. Reliability and Safety

It's so easy to program for the "sunny day".

  • The data is never bad. No fields have blanks. No fields have accented characters. All connected systems are functional.
  • There is no one trying to intentionally break the system.

Considerations:

  • How does the self-driving car operate at night? In a rainstorm? In a snowstorm?
  • How does the military unmanned drone operate when the GPS is down?
  • How does the self-driving car operate when an object is approaching from the side instead of the front or back?

3. Privacy and Security

  • Many countries and regions in the world are developing new standards and laws to protect the data of citizens.
  • Laws are always slower than technology.

4. Inclusiveness

"At Microsoft, we firmly believe everyone should benefit from intelligent technology, meaning it must incorporate and address a broad range of human needs and experiences."

  • How does a voice assistant work for someone who has a speech impediment?
  • How does a fitness tracker work for someone with a mobility disability?
  • How does "video-only" product instructions help someone who is blind?

5. Transparency

"When AI systems are used to help inform decisions that have tremendous impacts on people's lives, it is critical that people understand how those decisions were made."

  • "When someone is rejected by an AI system for a job, life insurance, or a bank loan, why were they rejected?
  • If you are unable to tell them why, the system lacks transparency and you can't tell if it's operating properly.
  • It could be unfair, and you won't know it."

6. Accountability

"The people who design and deploy AI systems must be accountable for how their systems operate."

  • AI systems should not be the "final authority" in decisions that significantly affect employment, finances, health care, and human safety.
  • There should be regular review of how the AI is operating.
  • Continuous improvement of the model is essential.

Fundamental Principles of Machine Learning on Azure (15–20%)

Common Machine Learning Types

  • Regression (supervised learning)
  • Classification (supervised learning)
    • Binary classification → only two answers (0 and 1)
    • Multi-class classification → allows for multiple options
  • Clustering (unsupervised learning)

Machine Learning vs Deep Learning

Machine Learning

Process: Input → Feature Extraction → Classification → Output

Deep Learning

Process: Input → [Multiple interconnected layers] → Output

Combines Feature Extraction and Classification within the same model

Transformer Architecture

Foundation Models

GPT BERT T5

Key Components

  • Encoder → Takes input and converts into tokens
  • Decoder → Generates output (used in text generation)
  • Attention Mechanism → Decides which tokens get higher or lower weightage

Benefits

  • Can handle long input text
  • Understands context and relationships between words
  • Enables faster training using parallel processing
  • Powers most state-of-the-art transformer models (e.g., GPT, BERT)

Core Machine Learning Concepts

Features & Labels

  • Features → Input variables
    • Example: Date of Birth, Gender, Smoker
  • Label → The thing we're predicting
    • Example: Life expectancy

Training Model

  • Training data
  • Validation data

Confusion Matrix

Definition: Comparison of what the model predicted vs. what actually happened.

Predicted Actually
True Positive Yes Yes
True Negative No No

Evaluation Metrics

Accuracy
Accuracy = Correct Predictions / Total Predictions
Precision
Precision = Correct Positives / All Predicted Positives

F1 Score

Definition: Combines Precision and Recall into a single metric.

Classification Models are Tested Using:

  • Test Data
  • Confusion Matrix
  • Evaluation Metrics

Key Considerations

  • Accuracy alone is not enough
  • Good evaluation = Technical metrics + Business context
