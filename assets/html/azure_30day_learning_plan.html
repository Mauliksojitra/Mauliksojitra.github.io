CI Pipeline - Build Automation

Create automated build pipeline

Sample YAML Pipeline:

trigger:
  branches:
    include:
      - main
      - develop

pool:
  vmImage: 'ubuntu-latest'

variables:
  buildConfiguration: 'Release'

stages:
- stage: Build
  jobs:
  - job: BuildJob
    steps:
    - task: NodeTool@0
      inputs:
        versionSpec: '18.x'
      displayName: 'Install Node.js'
    
    - script: |
        npm install
        npm run build
      displayName: 'Install and Build'
    
    - task: CopyFiles@2
      inputs:
        Contents: 'dist/**'
        TargetFolder: '$(Build.ArtifactStagingDirectory)'
    
    - task: PublishBuildArtifacts@1
      inputs:
        PathtoPublish: '$(Build.ArtifactStagingDirectory)'
        ArtifactName: 'drop'

CD Pipeline - Release Management

Implement multi-stage deployments

Multi-Stage YAML Deployment:

stages:
- stage: Deploy_Dev
  jobs:
  - deployment: DeployWeb
    environment: 'development'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: AzureWebApp@1
            inputs:
              azureSubscription: 'Azure-Connection'
              appName: 'myapp-dev'
              package: '$(Pipeline.Workspace)/drop/*.zip'

- stage: Deploy_Staging
  dependsOn: Deploy_Dev
  jobs:
  - deployment: DeployStaging
    environment: 'staging'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: AzureWebApp@1
            inputs:
              azureSubscription: 'Azure-Connection'
              appName: 'myapp-staging'
              package: '$(Pipeline.Workspace)/drop/*.zip'

- stage: Deploy_Production
  dependsOn: Deploy_Staging
  condition: and(succeeded(), eq(variables['Build.SourceBranch'], 'refs/heads/main'))
  jobs:
  - deployment: DeployProd
    environment: 'production'
    strategy:
      runOnce:
        deploy:
          steps:
          - task: AzureWebApp@1
            inputs:
              azureSubscription: 'Azure-Connection'
              appName: 'myapp-prod'
              package: '$(Pipeline.Workspace)/drop/*.zip'

Tasks:

  1. Create environments: Development, Staging, Production
  2. Configure approval gates on Production environment
  3. Add deployment stages in YAML pipeline
  4. Configure Azure service connection (authorize to subscription)
  5. Deploy to App Service in each environment
  6. Add pre-deployment and post-deployment conditions
  7. Configure deployment strategy: rolling, blue-green, canary
  8. Set up manual intervention for production
  9. Monitor deployment progress

Key Concepts: Environments, approvals, deployment strategies, service connections
